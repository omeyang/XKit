# 01-例外决策记录规范

本文件约定在 XKit 项目中记录偏离 Constitution 的例外决策方式，确保所有例外都有清晰的理由、边界和回退路径。

## 1. 适用范围

- 所有明确偏离 `.specify/memory/constitution.md` 中规定的原则或约束的情况。
- 例外类型包括（但不限于）：
  - 临时放宽测试覆盖率或性能要求；
  - 临时引入不符合依赖管理规范的第三方库；
  - 临时关闭部分质量门禁检查。

## 2. 记录内容

每条例外记录至少包含：

- 背景与原因：说明为什么需要例外；
- 影响范围：涉及的模块、服务或 Feature；
- 风险评估：对稳定性、安全性和维护成本的影响；
- 缓解措施：降低风险的具体做法；
- 有效期：例外生效的起止时间或触发结束条件；
- 回退计划：恢复合规后的具体操作。

## 3. 记录方式

- 例外记录以追加方式维护在本文件中，采用倒序或分章节组织均可，但必须保持单条记录职责清晰。
- 与具体 Feature 强相关的例外，可以在对应 Feature 的 `decisions/` 或 `adr/` 中留存详细记录，本文件保留引用和摘要。

## 4. 审批与复盘

- 例外必须经过约定角色审批（如架构负责人或团队负责人）后才可执行。
- 例外到期后，需要：
  - 按计划恢复合规实现；
  - 在本文件中补充实际结束时间与复盘结论。

**[01D-requirement-levels.md](./01D-requirement-levels.md)** - 附录D：RFC 2119 标准（🆕 含使用指南）

包含文档要求表述的标准词汇（遵循 RFC 2119 国际标准）：
- **标准词汇定义**：
  - 强制性要求（"必须" MUST、"禁止" MUST NOT）
  - 强烈建议（"应该" SHOULD、"不应该" SHOULD NOT）
  - 可选建议（"可以" MAY）
- **RFC 2119 使用指南**（🆕 重要章节）：
  - 谨慎和节制原则（"must be used with care and sparingly"）⭐
  - 安全考虑（违反后果说明、检查方法）⭐
  - 标准引用声明（中英文模板）
  - 大小写约定（英文大写、中文加粗）
  - 语境修饰（文档类型影响强制性）
- **现有原则标注**：12个核心原则的强制性级别
- **例外情况规范**：偏离"应该"时的说明要求

**[01E-chinese-writing-style.md](./01E-chinese-writing-style.md)** - 附录E：中文技术写作规范

## ✅ 核心要求速查

### 文档结构

| 要求 | 标准 | 参考 |
|------|------|------|
| 文档编号 | 文件名以编号开头（如 `01-xxx.md`） | 原则 9 |
| 单一职责 | 文档标题能用一句话概括 | 原则 0 |
| 文档长度 | 单个文档 ≤ 800 行 | 附录C §6 |
| 章节层级 | 不超过 4 级（H1-H4） | 附录C §5 |

### 内容质量

| 要求 | 标准 | 参考 |
|------|------|------|
| 代码块 | ≤ 5 行，超过改用 Mermaid | 原则 1 |
| 表格列数 | ≤ 5 列 | 附录C §4 |
| 专业表述 | 删除口语化和自问自答 | 原则 3 |
| 数据来源 | 标注来源、时间、查询方法 | 原则 4 |
| 引用准确 | 所有链接和代码位置有效 | 原则 8 |

### 环境适配

| 要求 | 标准 | 参考 |
|------|------|------|
| 同架设计 | SaaS 和本地化使用同一代码 | 原则 2 |
| 差异标注 | 仅标注环境差异，不全量对比 | 附录C §3 |
| 配置开关 | 代码中体现配置切换方式 | 附录C §3 |

### 历史信息

| 要求 | 标准 | 参考 |
|------|------|------|
| 只记录最新 | 文档只描述当前状态 | 原则 7 |
| 历史单独管理 | ADR 或 archived 目录 | 原则 7 |
| 决策留痕 | 记录已拒绝方案 | 原则 11 |

### 代码溯源

| 要求 | 标准 | 参考 |
|------|------|------|
| 代码基准 | 标注仓库、分支、提交、日期 | 原则 10 |
| 代码位置 | `文件:行号` 格式 | 附录C §1 |
| Worktree 标注 | 明确标注工作树名称 | 原则 10 |

## 模块职责与审查要点

### xconf — 配置加载与热重载

**职责**：YAML/JSON 格式自动检测、配置加载、文件 Watch 热重载、底层 koanf 暴露。

**重点审查**：

#### 加载与解析
- [ ] **格式检测**：YAML 和 JSON 的自动检测依据是什么（扩展名？内容嗅探？）？混合格式文件是否有明确错误？
- [ ] **类型映射**：配置值到 Go 类型的映射是否严格？数字字符串是否意外转为 int？
- [ ] **必选项校验**：是否支持标记必选配置字段？缺失时是否 fail-fast？
- [ ] **默认值**：是否支持在 struct tag 或代码中声明默认值？默认值与文件值的优先级？
- [ ] **环境变量覆盖**：是否支持环境变量覆盖文件配置？覆盖规则（前缀、分隔符）是否清晰？

#### 热重载（Watch）
- [ ] **防抖机制**：文件修改事件的防抖时间窗口是否可配置？默认值是否合理？
- [ ] **原子性**：重载过程中读取配置是否可能读到中间状态？是否有原子切换机制？
- [ ] **回调通知**：重载后是否通知订阅者？回调函数 panic 是否影响主流程？
- [ ] **错误处理**：重载失败（文件格式错误、权限问题）时是否保留旧配置？错误是否有日志/回调？
- [ ] **goroutine 管理**：Watch goroutine 是否随 `Close()` 正确退出？是否有 goroutine 泄漏风险？
- [ ] **文件删除**：Watch 的文件被删除时的行为？崩溃？忽略？切换到旧值？

#### API 设计
- [ ] **Client() 暴露**：暴露底层 koanf 实例是否合适？调用方直接操作 koanf 可能绕过热重载通知
- [ ] **并发读取**：多个 goroutine 同时读取配置是否安全？底层 koanf 是否线程安全？
- [ ] **Unmarshal 接口**：是否支持将配置段反序列化为任意 struct？tag 格式是否与 koanf 一致？

#### 安全性
- [ ] **路径穿越**：配置文件路径是否经过 `xfile.SanitizePath` 校验？
- [ ] **敏感信息**：密码、token 等敏感配置是否有专门处理（加密存储、日志脱敏）？

---

## 跨包一致性检查

- [ ] xconf 的热重载回调与 xlog 的动态级别调整是否集成？配置变更是否能自动刷新日志级别？
- [ ] xconf 加载的配置路径是否使用 xfile 进行安全校验？
- [ ] xconf 是否与 xenv 的部署类型联动（如不同环境加载不同配置文件）？

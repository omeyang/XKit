## 模块职责与审查要点

### xauth — 认证服务客户端

**职责**：与认证服务交互，提供 token 验证、用户信息获取等能力。

**重点审查**：

#### 安全性（认证模块最核心的属性）
- [ ] **Token 验证**：验证逻辑是否完整（签名、过期时间、issuer、audience）？
- [ ] **Token 泄露防护**：token 是否在日志、错误信息、panic stack 中脱敏？
- [ ] **时间安全**：token 过期判断是否考虑时钟偏移（允许几秒的 leeway）？
- [ ] **HTTPS 强制**：与认证服务的通信是否强制 HTTPS？是否有 TLS 验证？

#### 缓存策略
- [ ] **缓存命中率**：认证结果缓存（xlru）的 TTL 和容量是否合理？
- [ ] **缓存失效**：token 被撤销时缓存如何失效？是否有主动失效机制？
- [ ] **缓存穿透**：无效 token 是否缓存（防止恶意请求击穿到认证服务）？
- [ ] **缓存一致性**：缓存中的用户信息过期策略？用户信息变更后缓存多久更新？

#### 错误处理
- [ ] **认证服务不可用**：服务不可用时是否有降级策略（如使用缓存的验证结果）？
- [ ] **错误分类**：401（未认证）和 403（未授权）是否明确区分？
- [ ] **重试策略**：网络超时是否重试？重试是否使用 xretry？

#### API 设计
- [ ] **Context 集成**：认证信息是否通过 xctx 注入 context？
- [ ] **中间件**：是否提供 HTTP/gRPC 认证中间件？中间件链中的位置？
- [ ] **与 xtenant 关系**：认证后的租户信息是否与 xtenant 一致？

### xrun — 进程生命周期管理

**职责**：统一管理服务启动、健康检查、优雅关闭。

**重点审查**：

#### 启动流程
- [ ] **启动顺序**：依赖组件的启动顺序是否可控（如先连数据库再启动 HTTP server）？
- [ ] **启动失败**：某个组件启动失败时是否回滚已启动的组件？
- [ ] **就绪检查**：是否支持 readiness probe（K8s）？所有依赖就绪后才标记为 ready？

#### 优雅关闭（Graceful Shutdown）
- [ ] **信号处理**：是否监听 SIGTERM/SIGINT？是否可配置监听的信号？
- [ ] **关闭顺序**：关闭顺序是否为启动顺序的逆序？是否可自定义？
- [ ] **关闭超时**：是否有全局关闭超时？超时后是否强制退出（os.Exit）？
- [ ] **在途请求**：HTTP server 关闭时是否等待在途请求完成？
- [ ] **后台任务**：goroutine pool（xpool）、cron（xcron）是否在关闭时正确停止？
- [ ] **资源释放顺序**：DB 连接、Redis 连接、etcd session 的释放顺序？

#### 健康检查
- [ ] **存活检查（Liveness）**：检查进程本身是否健康（无死锁、goroutine 不泄漏）
- [ ] **就绪检查（Readiness）**：检查所有依赖是否可用（DB、Redis、MQ）
- [ ] **依赖健康聚合**：是否聚合所有依赖的健康状态？一个依赖不健康时整体状态？
- [ ] **HTTP endpoint**：健康检查是否暴露 `/healthz` 和 `/readyz` 端点？

#### 钩子机制
- [ ] **生命周期钩子**：是否支持 OnStart/OnStop 钩子？钩子执行失败的影响？
- [ ] **钩子超时**：每个钩子是否有单独的超时控制？
- [ ] **钩子并行**：无依赖的钩子是否可以并行执行以加快启动/关闭？

### xdbg — 运行时调试服务

**职责**：运行时调试能力（如 pprof、动态配置修改等）。

**重点审查**：

#### 安全性（调试服务的最大风险）
- [ ] **访问控制**：调试端点是否有认证/鉴权？生产环境是否可以禁用？
- [ ] **端口隔离**：调试服务是否使用独立端口（不与业务 API 共享）？
- [ ] **信息泄露**：pprof 输出是否可能暴露敏感信息（内存中的密码、token）？
- [ ] **动态配置风险**：运行时修改配置是否有审计日志？修改范围是否受限？

#### 功能完整性
- [ ] **pprof 集成**：CPU profile、Heap profile、Goroutine stack 是否完整暴露？
- [ ] **日志级别**：是否支持运行时修改日志级别（集成 xlog 动态级别）？
- [ ] **Metrics dump**：是否支持导出当前 metrics 快照？
- [ ] **配置查看**：是否支持查看当前生效的配置（脱敏后）？

#### 生命周期
- [ ] **启动条件**：是否可通过配置或环境变量控制是否启动调试服务？
- [ ] **关闭**：是否与 xrun 集成，在进程关闭时正确停止调试服务？

## 跨包一致性检查

- [ ] xauth 的 context 操作是否使用 xctx 的标准 API（WithXxx / Xxx）？
- [ ] xauth 的 metrics 是否使用 xmetrics 接口？
- [ ] xrun 的关闭流程是否覆盖所有 XKit 组件（xdlock session、xcron scheduler、xkafka consumer 等）？
- [ ] xdbg 的日志级别修改是否直接调用 xlog 的 API？
- [ ] 三个包的 Options 模式是否与其他 XKit 包一致？
- [ ] xrun 是否与 xenv 集成（如根据部署类型调整行为）？

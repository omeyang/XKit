# 01 · 总体目标

## 设计目标

XKit 是独立的 Go 基础工具库，为业务服务提供通用能力（可观测性、弹性、存储、消息、分布式协调等）。

- **模块化**：每个包职责单一、可独立引入，包间依赖显式最小化。
- **API 稳定**：公开 API 遵循语义化版本；破坏性变更需升 MAJOR 并记录 ADR。
- **零值可用优先**：公开类型在符合语义时默认零值可用，不能零值使用的类型须在类型注释中声明。
- **构造失败返回 error**：构造器不 panic（参见 ADR 0001）。
- **接口由使用方定义**：库内不强加接口；消费方按需定义（参见 ADR 0002）。

## 质量目标

| 指标 | 目标 |
|---|---|
| 核心逻辑测试覆盖率 | ≥ 95% |
| 整体测试覆盖率 | ≥ 90% |
| `golangci-lint` | 全量通过（配置见 `.golangci.yml`） |
| 函数长度 (`funlen`) | ≤ 70 行 |
| 圈复杂度 (`gocyclo`) | ≤ 10 |
| 数据竞争检测 (`-race`) | 全部测试通过 |
| 漏洞扫描 (`govulncheck`) | 无高危未处置项 |

## 性能目标

- **热路径零分配优先**：高频调用 API 使用基准测试锁定 allocs；回归需在 PR 说明。
- **并发安全显式声明**：类型是否协程安全在类型注释中声明；非安全类型禁止在 doc 示例中跨 goroutine 使用。
- **内存池化**：大对象/高频分配使用 `sync.Pool`，使用处注明回收责任。

## 文档目标

- 文档只描述**当前客观状态**；演进过程进 `CHANGELOG.md` 或 `docs/_archive/`。
- 每个包提供 `doc.go`；公开 API 有中文注释。
- 关键决策以 ADR 形式留痕，含被拒方案（参见 [05-decisions](05-decisions/00-index.md)）。

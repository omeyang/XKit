# 0006 · 中文文档注释 + 英文标识符

- **状态**：Accepted
- **范围**：项目级

## 背景

维护方为中文团队，文档阅读效率以中文最优；但代码标识符、文件名需要与 Go 生态一致以便工具链、IDE、外部协作。

## 决策

- **中文**：`doc.go` 包注释、公开 API 的 godoc 注释、`docs/` 下所有 Markdown、测试场景描述（`t.Run` 名可中文）。
- **英文**：包名、类型名、函数名、变量名、常量名、文件名、error sentinel 的 `err.Error()` 文案、日志 message（便于 grep 与全球化协作）。

## 备选方案（被拒）

- **全英文**：阅读/评审效率降低，长文档维护成本上升。
- **全中文（含标识符）**：违反 Go 社区惯例；非 ASCII 标识符在工具链、日志检索中易出问题。

## 影响

- **正向**：文档可读性好；代码仍可被开源协作。
- **代价**：评审需同时兼顾两种语言；对英文不熟的新人在写日志 message 时需要约束。

## 代码引用

- `pkg/*/doc.go`（中文包注释）
- 任意包的公开 API godoc（中文）
